Pyramid Web Framework

What is Pyramid?

Pyramid is a Web framework that was created in December of 2010. It is part of the Pylons Project and was a merger of Pylons and Repoze in which features were added and removed to make it more useful for Pylons framework. The concepts and features of Pyramid are inspired by Zope, Pylons, and Django. Specific things that were taken from each include:

  • Zope - extensibility, traversal, and declarative security
  • Pylons - URL dispatch, non-opinionated view of persistence, and templating
  • Django - view and level of documentation

Pyramid is considered to have a MVC (model-view-controller) framework, but it doesn't use the MVC terminology. For example, instead of having "controllers" (classes with methods that map to URLs), it has "views".

Pyramid has about 650 pages of thorough documentation on basic to advanced topics. It is also a great framework to use for getting started on a project quickly as you don't have to know a lot in order to start a project. Pyramid is also good for prototyping concepts as well as for developing either small or large web applications.

Tenants of Pyramid

  1. Simplicity - can use Pyramid and get results even if you only partially understand it because there are a minimal number of core concepts
  2. Minimalism - it provides fast solutions to fundamental activities of creating web applications
  3. Documentation - all aspects of Pyramid have up-to-date documentation
  4. Speed - it provides fast execution of common tasks
  5. Reliability - 100% of the content of Pyramid is tested
  6. Openness - it has a permissive open source license

System Requirements

  • Python 2.6 or newer
  • UNIX-like system (like Linux, MacOS X, and FreeBSD) or Windows platform
  • Python interpreter

How to Download

Pyramid is installed in python development mode and can be installed on a virtual environment. Instructions on how to download Pyramid on different systems or on a virtual environment can be found here. When Pyramid is installed, so are various libraries such as WebOb and PasteDeploy. Scaffolds are also registered, which make it easy to start a new Pyramid project.

Differences Between Pyramid and Pylons

Pyramid is a new framework that is different from the Pylons Web framework. However, Pylons 1.0 code will run inside Pyramid through a fallback handler. For those who currently use Pylons and may want to eventually switch to Pyramid, they should avoid using global objects except in action methods and ensure all routes are explicit and have a unique name.

Differences Between Pyramid and repoze.bfg

Pyramid is based off repoze.bfg, but it has a new name and some new features. These new features come from what was added from Pylons 1.0. Users who want to make the switch from repoze.bfg to Pyramid don't have to make any changes and can use an automated porting process to make the conversion.

More Useful Attributes of Pyramid

  • Faster than other Python frameworks
  • Has a "Cookbook" of demonstrations of common integration scenarios and Pylonsproject.org has tutorials and information
  • Can write single-file applications, which is great for debugging
  • Has a debugging toolbar as well as debugging settings that notify when something is wrong
  • Paster has templates that can quickly build the skeleton of the infrastructure for several types of Web applications
  • Can generate URLs
  • Can use many different databases including SQLAlchemy, Zope Object Database, and other NoSQL databases
  • Commits at the end of a request or aborts if there's an exception so bad changes or requests won't be committed
  • Can use add-ons that provide additional functionality
  • Can use extensible templating (including Mako, Genshi, Chameleon, and Jinja2) and can even use multiple templating systems in one project
  • Can use multiple Python Packages
  • Has a flexible pluggable authentication system
  • Has traversal with a tree of resources with different associated views (i.e. managers have a different view than employees)
  • Can subscribe to listen to Pyramid events or your own that you create
  • Composed of the best features from other Python Web frameworks
